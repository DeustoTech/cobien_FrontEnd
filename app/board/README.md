# Board Module

This package implements the "board" feature: a read-only message wall that shows
text + optional image content addressed to the current furniture device.

## Scope

- Render the board UI (`BoardScreen`) with navigation and delete actions.
- Fetch board messages from the backend API.
- Fallback to MongoDB/GridFS and finally to local cache when needed.
- Cache downloaded images and normalized message payloads for resilience.
- React to MQTT navigation/reload events.

## Files

- `boardScreen.py`
  - Kivy screen implementation and UI layout (KV string).
  - Message rendering, left/right navigation, deletion confirmation.
  - MQTT listener for board reload and "reload latest message" flows.

- `loadBoard.py`
  - Data access layer for board messages.
  - Primary source: REST API (`pizarra_messages_url`).
  - Fallback source: MongoDB collection `pizarra_messages` + GridFS bucket `pizarra_fs`.
  - Local cache persistence and image caching/orientation correction.

- `__init__.py`
  - Package marker.

## Runtime Data / Non-Python Assets

- `board_cache/`
  - Runtime cache folder generated by the application.
  - Contains:
    - Image binaries downloaded from API/GridFS (`<id>.<ext>`).
    - Index file `board_items.json` with normalized cached entries.
  - Safe to delete; it will be recreated automatically.

- `__pycache__/`
  - Python bytecode cache generated at runtime/development.
  - Not source code.

## Data Flow

1. `BoardScreen.refresh_from_mongo()` asks `fetch_board_items_from_mongo(...)`.
2. Loader tries API first and normalizes payload.
3. If API fails, loader queries MongoDB + GridFS.
4. If MongoDB fails, loader returns local cache.
5. Screen renders index `0` (latest message) by default.

## Configuration Dependencies

Read from unified config (`app/config/config.local.json` via `config_store`):

- `services.pizarra_messages_url`
- `services.pizarra_delete_url_template`
- `services.notify_api_key`
- `services.mongo_uri` (used by events/loadEvents client path)

Also uses:

- `settings.device_id` (recipient key via `AppConfig().get_device_id()`)
- MQTT local broker settings (`MQTT_LOCAL_BROKER`, `MQTT_LOCAL_PORT`)

## Notes

- Message deletion is best-effort and handled through the backend API helper.
- Image orientation is corrected using EXIF metadata when available.
- The board screen intentionally prioritizes reliability over strict source purity:
  API -> Mongo -> cache.

## Known Technical Debt / Bad Practices

- Large inline KV definitions in `boardScreen.py` make UI maintenance and review difficult.
- Logging is implemented with `print()` statements instead of structured `logging`.
- Broad `except Exception` blocks reduce observability and hide concrete failure causes.
- `loadBoard.py` combines transport, storage, transformation, and cache concerns in one module.
